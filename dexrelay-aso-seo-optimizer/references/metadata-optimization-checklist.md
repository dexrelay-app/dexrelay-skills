# Metadata Optimization Checklist

Use this before finalizing App Store or TestFlight copy.

## Title

- fits within 30 characters
- includes the strongest core concept naturally
- keeps the brand readable
- avoids filler punctuation and wasted words
- does not sound like a keyword dump

## Subtitle

- fits within 30 characters
- adds new meaning instead of repeating the title
- explains the next-most-important benefit or job
- stays readable in isolation

## Keyword Field

- 100 characters or fewer
- comma-separated with no spaces
- does not repeat title or subtitle words
- avoids competitor brands
- avoids `app`, `free`, and obvious waste
- prioritizes relevance before breadth

## Promotional Text

- 170 characters or fewer
- explains what is new, urgent, or differentiated
- can change without review, so use it for launches and updates

## Description

Prioritize conversion over stuffing:

1. first 2 to 3 lines explain the core outcome
2. proof or trust signal appears early
3. features are framed as benefits
4. formatting is skimmable
5. ends with a clear call to action

## TestFlight Surfaces

- beta app description is shorter and more direct than the App Store description
- `What to Test` focuses on the current unstable or high-value flows
- release notes explain meaningful user-facing changes only

## Failure Modes To Flag

- repeated keywords across title, subtitle, and keyword field
- generic phrases like `AI assistant` without a specific job
- title optimized for search but too unclear to convert
- subtitle that lists features instead of outcomes
- descriptions that start with `Welcome to`
- metadata that promises features not actually shipped
